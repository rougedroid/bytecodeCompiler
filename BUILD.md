# Building and Running the ByteCode Compiler

This document provides detailed instructions for building and running the ByteCode Compiler from source.

## Prerequisites

- **GCC** or compatible C compiler (gcc 5.0+)
- **C Standard Library** (glibc or equivalent)
- **Unix-like environment** (Linux, macOS, BSD, or Windows with WSL/Cygwin)

No external dependencies or libraries are required.

## Build Instructions

### Basic Build

```bash
gcc -std=c99 -o compiler_main compiler_main.c
```

This compiles `compiler_main.c` using the C99 standard and produces an executable named `compiler_main`.

### Build with Debugging Symbols

```bash
gcc -std=c99 -g -o compiler_main compiler_main.c
```

The `-g` flag includes debugging symbols, allowing you to use GDB or other debuggers:

```bash
gdb ./compiler_main
```

### Build with Warnings Enabled

```bash
gcc -std=c99 -Wall -Wextra -o compiler_main compiler_main.c
```

This enables all standard compiler warnings, useful for catching potential issues.

### Build with Optimization

```bash
gcc -std=c99 -O2 -o compiler_main compiler_main.c
```

The `-O2` flag enables level-2 optimizations for faster execution (at the cost of longer compile time and larger binary).

## Running the Compiler

### Basic Execution

```bash
./compiler_main
```

This runs the compiler with the hard-coded program embedded in `load_code()`. Output is printed to `stdout`.

### Capturing Output

```bash
./compiler_main > output.txt
```

Redirects bytecode output to a file.

### Running with Error Stream Separate

```bash
./compiler_main 2>&1 | tee output.log
```

Captures both stdout and stderr to a log file while displaying in the terminal.

## Troubleshooting

### Compilation Errors

**Error: `gcc: command not found`**
- Install GCC: `sudo apt-get install gcc` (Ubuntu/Debian) or `brew install gcc` (macOS)

**Error: `implicit declaration of function`**
- Ensure you're using `-std=c99` flag to enable C99 standard features
- Check that all #include statements are present

**Error: `undefined reference to 'malloc'`**
- Ensure `-lc` is linked (should be default)

### Runtime Errors

**Segmentation fault (SIGSEGV)**
- Check that the input program in `load_code()` is valid syntax
- Memory corruption may have occurred; refer to memory handling code

**Unexpected output or incorrect bytecode**
- Verify the input program syntax matches [SYNTAX.md](SYNTAX.md)
- Add debug output by modifying the compiler source
- Check token classification in `classifier()` function

### Building on Different Platforms

**Windows (with MinGW):**
```bash
gcc -std=c99 -o compiler_main.exe compiler_main.c
compiler_main.exe
```

**macOS:**
```bash
gcc -std=c99 -o compiler_main compiler_main.c
./compiler_main
```

**Linux (various distributions):**
Same as basic build instructions above.

## Development Notes

- The compiler is a single-file project for educational simplicity
- No make file is provided; the compilation command is intentionally kept simple
- Memory debugging can be done with `valgrind`:
  ```bash
  valgrind --leak-check=full ./compiler_main
  ```
